GraphQL을 사용하는 이유

기능 단위로 API를 짜다보면 
화면 하나에 너무 많은 API를 호출해야하는 문제가 생긴다.
user/id
user/id/post
user/id/followers

한 번에 보내주려고 해도, 어떤 것이 들어가야하는지를 알아야 하는
커뮤니케이션 비용이 들어가게 된다.

그리고 기존 api를 수정한다면 거기에서 생기는 호환성 문제도 생긴다.

기획이 바뀌고 UI가 바뀌면 결국 API도 바뀌어야 하는데,
이렇게 되면 위의 말처럼 커뮤니케이션 비용이 증가하고
개발 프로세스가 둔화되는 문제가 생긴다.

기존 API 수정 최소화
신규 API 추가 최소화
커뮤니케이션 비용 최소화

이것을 해결할 수 있는 방법이 GraphQL이다.

GraphQL은 클라이언트가 조금의 문법과 데이터의 구조를 알면
쿼리를 가지고 데이터를 자유롭게 얻어올 수 있다.

대부분의 언어를 지원하고 있으며
특정 데이터베이스에도 종속되지 않는다는 점,
자체 디버깅 툴로 편리하게 관리할 수 있다는 점.
REST API와 함께사용할 수 있다는 점이 GraphQL의 특징이다.
